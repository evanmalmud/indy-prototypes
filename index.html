INDY PROTOTYPES

Three design pillars. One shared toolset. Turn-based, puzzle-first.

Every prototype below uses the same four tools — whip, torch, revolver, sand satchel — resolved through the same interaction table in src/kernel/tools.ts. That is deliberate: holding the verbs constant is what makes the three-way comparison isolate the pillar instead of confounding it with different mechanics.

The three pillars

  • A — “Clever, not strong”

    Static tomb, perfect information, no pressure. Pure deliberation: the room waits as long as you need.

  • B — “The tomb is alive”

    The room reacts and escalates every turn — collapsing floors, rising water, spreading fire, boulders in motion. Still fully turn-based and deterministic; the board changes under you.

  • C — “Greed is the real enemy”

    Tool uses cost scarce resources — torch fuel, bullets, sand — and treasure sits off the safe path. The puzzle is affording what you want.

The designed “aha” set

Combination What the player discovers
WHIP + BOULDER You can pull. Block puzzles assume push-only; inverting that re-opens every earlier room. The flagship.
SAND + PRESSURE PLATE Weight substitution — the literal Raiders idol swap.
TORCH + OIL TRAIL Fire propagates one tile per turn, so time becomes a puzzle element rather than just a resource.
REVOLVER + ROPE Sever at range — you are aiming the falling payload.
WHIP + TORCH Fling a lit torch across a gap. Remote ignition.
REVOLVER + GAS VENT Spark → explosion. The safe tool is the wrong tool here.
